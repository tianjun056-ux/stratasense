## L1–L5 分层模型（是什么，不是什么）

> **L1–L5 是认知分层模型，不是工程模块，也不是数据管道。**

| 层级 | 含义 | 时间尺度 | 在本工具中的作用 |
|---|---|---|---|
| **L1 世界结构层** | 制度 / 地缘 / 文明规则 | 5–10 年 | 决定“看世界的方式” |
| **L2 资源与瓶颈层** | 能源 / 矿产 / 算力 / 产能 | 2–5 年 | 识别不可替代约束 |

| **L5 执行层** | 交易 / 行动 | 即时 | **不在 StrataSense 内** |

**重要澄清：**
- L1–L5 是你用来“理解世界”的**认知坐标系**
- 它们不对应代码目录

## 这是什么

**StrataSense** 是一个用于**冻结与对比你“世界认知结构”**的工具。

它不追踪新闻、不分析市场、不输出观点，而是：

- 明确你**承认哪些信息源有资格进入你的长期世界模型**
- 将这些认知结构**固化为可审计的快照**
- 在你**主动修改结构时**，通过 diff 明确指出“哪里变了”

你可以把它理解为：

> **世界结构的 Git + diff，而不是信息流工具。**

---

## 明确不做什么（这是边界）

StrataSense **刻意不做**以下事情：

- ❌ 不接 RSS / API  
- ❌ 不抓取实时新闻  
- ❌ 不生成投资 / 决策建议  
- ❌ 不做情绪或热点聚合  
- ❌ 不“自动学习”或“智能判断”

> 如果你在寻找“多看点信息”的工具，  
> **StrataSense 不是。**

## 核心设计原则

### 1️⃣ 冷存优先（Cold-First）
- 99% 的输出进入 `outputs/`
- 不主动打断用户

### 2️⃣ 默认沉默（Default Silence）
- 定时运行 **不通知**
- 只有结构性变化才通知

### 3️⃣ 人工可控摩擦
- 结构信息源 **写死在代码中**
- 修改必须经过：
  - 明确编辑
  - Git 提交
  - diff 对比

这是**刻意的“慢”**，用于防止认知被动态污染。

## 输出说明

每次运行会生成：

```
outputs/
├── latest/
│   ├── report.json
│   ├── report.md
│   ├── diff.json        # 若存在结构变化
│   └── diff.md
└── runs/
    └── run_YYYYMMDD_HHMMSS/
        ├── report.json
        ├── report.md
        ├── diff.json
        └── diff.md
```

### diff 的含义

* **added**：你新增承认的信息源
* **removed**：你不再承认的信息源
* **unchanged**：结构未变

> diff 表示的是：
> **“你的世界认知结构是否发生了变化”**
> 而不是“今天发生了什么新闻”。

## 通知策略（已内置）

* **定时运行（GitHub Actions）**

  * ❌ 无变化 → 不通知
  * ✅ 有结构变化 → PushDeer 通知

* **手动触发（workflow_dispatch）**

  * ✅ 一定通知
  * 用于确认系统健康状态

## 适合谁

StrataSense 适合：

* 希望**降低信息噪音**的人
* 有长期世界观 / 宏观框架的人
* 不希望被工具“牵着走”的人
* 希望对自己认知变化**可审计、可回溯**的人

不适合：

* 想要实时资讯
* 想要自动推荐
* 想要“帮我做决策”的系统
